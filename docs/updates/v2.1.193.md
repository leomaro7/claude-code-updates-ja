# Claude Code v2.1.193 アップデートまとめ

> 出典: https://code.claude.com/docs/en/changelog#2-1-193

## 💡 注目ポイント

### 1. Autoモードのシェルコマンド分類が強化 — すべてのBash/PowerShellコマンドが自動分類対象に

これまでAutoモードは「任意のコード実行パターン」だけを分類対象としていましたが、今回の変更でBash/PowerShellコマンド全体が分類対象になりました。これにより、ユーザーが意図せず危険なコマンドを実行するリスクがさらに低減されます。`autoMode.classifyAllShell`設定を有効にすると、すべてのシェルコマンドがAutoモード経由で安全チェックを受けるようになります。

### 2. Autoモード拒否理由の可視化 — `/permissions`で拒否履歴を確認可能に

Autoモードによるコマンド拒否時に表示されるトースト通知やトランスクリプトに、拒否された理由が明示されるようになった。さらに`/permissions`コマンドで最近の拒否履歴を確認できるため、ユーザーはなぜコマンドが拒否されたのかを理解し、必要に応じて設定を調整しやすくなります。

### 3. オープンテレメトリー対応の強化 — アシスタント応答のログ記録が追加

`claude_code.assistant_response`という新しいOpenTelemetryログイベントが追加され、アシスタントの応答テキストが記録されるようになった。ただし、デフォルトではマスク処理され、`OTEL_LOG_ASSISTANT_RESPONSES=1`環境変数を明示的に設定しない限り、ユーザー入力のみが記録されます。既存の環境変数設定と連携しているため、`OTEL_LOG_USER_PROMPTS`を有効にしている環境では、アップグレード後に自動的に応答も記録されるようになります。
