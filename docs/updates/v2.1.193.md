# Claude Code v2.1.193 アップデートまとめ

> 出典: https://code.claude.com/docs/en/changelog#2-1-193

## 💡 注目ポイント

### 1. Autoモードのシェルコマンド分類が強化 — すべてのBash/PowerShellコマンドが自動分類対象に

これまでAutoモードは「任意のコード実行パターン」のみを分類対象としていましたが、今回の変更で`autoMode.classifyAllShell`設定を導入し、すべてのBash/PowerShellコマンドが分類対象になりました。これにより、安全なコマンドとリスクのあるコマンドをより正確に区別できるようになり、不要な実行を防ぐことが可能に。

### 2. Autoモード拒否理由の可視化 — `/permissions`で最近の拒否内容を確認可能に

Autoモードによるコマンド拒否時に表示されるトースト通知やトランスクリプトに、拒否された理由が明示されるようになった。さらに`/permissions`コマンドで最近の拒否内容を確認できるため、ユーザーはなぜコマンドが拒否されたのかを理解し、適切な対応を取れるようになった。

### 3. オープンテレメトリー対応の強化 — アシスタント応答のログ記録機能を追加

`claude_code.assistant_response`という新しいOpenTelemetryログイベントを追加し、アシスタントの応答テキストを記録できるようにした。`OTEL_LOG_ASSISTANT_RESPONSES=1`を設定することで応答内容をログに記録でき、`OTEL_LOG_USER_PROMPTS`が有効な環境ではデフォルトで有効になる（`OTEL_LOG_ASSISTANT_RESPONSES=0`で無効化可能）。

### 4. バックグラウンドタスク管理の改善 — 偽りのタスク放棄通知を解消

バックグラウンドタスクの管理に関する複数のバグを修正した。これにより、セッション移行時の「Nつのバックグラウンドタスクが放棄されます」という誤った通知が解消され、すべてのタスクが新しいセッションに正しく引き継がれるようになった。さらに、メインターンのバックグラウンド化時に不要なサブエージェントが生成される問題も修正された。
